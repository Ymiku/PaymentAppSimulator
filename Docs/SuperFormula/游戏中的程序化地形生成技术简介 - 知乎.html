摘要

程序化地形生成技术可以帮助游戏开发人员快速地创建地形，建立游戏世界的基本面貌，加快迭代开发的过程。本文将以作者个人的经验和理解为基础，结合网络上的相关文档，一步一步地从基础开始简要地介绍程序化地形生成技术在游戏中的实现，希望能对初学者有所帮助。


  1. 高效的伪随机数生成器

正所谓数学是一切科学的根基，要实现程序化生成地形，也不例外。正如你可能玩过的很多有趣的游戏一样，程序生成的游戏世界诞生于一个随机种子，这个随机种子决定了整个世界的面貌，因此一个好的、高效的伪随机数生成器是非常重要的。

我们将为游戏世界中的每个位置都计算若干次随机数来决定这个位置是什么、有多高、或者密度等信息，而我们做这样的计算时往往只能知道当前位置的坐标信息，不能得到其它地点的信息（否则整个算法必须是顺序相关的，并且会丢失后文将提到的许多好的特性），我们需要伪随机数生成器具备这样的性质：

  • 只跟坐标位置相关
  • 简单快速

也就是：

一个来自已找不到链接的网页里的参考实现如下（感谢原作者）：

float Random(uint x, uint y, uint z, uint p0, uint p1, uint p2, uint p3)

{

uint n = x + p0 * y + p1 * z;

n = (n<<13) ^ n;

return (n * (n * p2) + p3) / 4294967296.0;

}

其中（p0, p1, p2, p3）为四个质数，并且其位数建议分别为（10，15，20，30），而（x, y, z）则是正整形的位置坐标。此函数实际上是正整数空间到一个随机数空间的映射。函数虽简单，但它的随机性却是可以满足需求的，其中的原理还请各位数学大牛洞悉。除此之外，肯定还存在能满足我们需求的更好的函数，但就本文中的例子而言，这已经足够。

有了它，我们就可以通过指定一个质数集合（即种子）来开始生成整个世界了。


2. 分形

我们想生成尽可能真实的地形，而不是一堆噪音。分形理论告诉我们真实世界普遍具有分形结构，地形当然是其中最典型的一种。考察最简单的一维地形高度图，由若干个不同频率和振幅的波形可以叠加成一个看似自然的、复杂无规律的高度图，这其实就是经典的Perlin分形噪音[1]。同样的原理可以应用到2D、3D或者任意维度上。

3. 无限大的高度图地形

前文介绍的伪随机数生成器有一个很好的性质，那就是只跟坐标位置相关，利用这个性质，我们可以很容易地生成无限大的地形，因为只要给出一个点的坐标，我们就可以计算出它的地形信息，而跟其它位置无关。这样的好处是显而易见的：

  • 地形的生成是顺序无关的
  • 甚至可以不保存地形数据，需要时计算一下即可，节省大量的存储空间
  • 可以并行计算

高度图其实是一个2D的结构，那么对于给定的一个坐标点（x, y），我们的高度图函数需要返回一个高度值h：

我们可以想象在整个正整数空间里存在若干条不同波长（频率）的波形，对于每条波形，其上的每个点的值能通过对其附近的波长对齐的点值插值得到，在2D高度图这个例子中，我们使用的是Bilinear插值：

float F(uint wave_length, uint x, uint y, vec seed_i){

uint x_align = x / wave_length;

uint y_align = y / wave_length;

float x_l = (float)(x_align % wave_length) / wave_length;

float y_l = (float)(y_align % wave_length) / wave_length;

float r00 = Random(x_align + 0, y_align + 0, seed_i);

float r01 = Random(x_align + 0, y_align + 1, seed_i);

float r10 = Random(x_align + 1, y_align + 0, seed_i);

float r11 = Random(x_align + 1, y_align + 1, seed_i);

return lerp(

lerp(r00, r01, y_l),

lerp(r10, r11, y_l),

x_l);

}


注意对于3D的情况，则需要对附近的8个点进行Trilinear插值。

最后把各个频率的波形叠加起来，就生成了类似这样的高度图：



图2. 分形噪音高度图与其对应的Mesh。http://paulbourke.net/fractals/noise[2]

但感觉还差点什么，真实的地形不会一直都是那么起伏不平的。为了区分平原和山地，让地形看起来更自然，我们可以把高度小于某个阙值的点当作平地。更进一步，为了让平原也有些许起伏，我们可以利用类似的分形方法生成平原的高度图，只不过振幅更小，也就是分别对平原和山地计算高度图，然后取两者的最大值混合起来，地形看上去就会自然许多：

图3. 对平原和山地分别计算高度图再取最大值的效果

4. 山洞

到目前为止我们的地形本质上还是2D的，因为只是一张高度图而没有内在结构。为了实现山洞、悬空结构等地形，我们需要一个3D的分形函数来告诉我们某个坐标点到底是空气还是地形。

另外一个难点是可视化，即如何生成可渲染的三角形网格。我们知道高度图是可以比较容易地三角形化的[3]，但一旦扩展到3D结构，我们就需要跟Voxel打交道了。类似的，我们可以用一个3D数组来保存地形信息，每个数组元素（Voxel）保存的不再是高度值，而是一个近似的距离值（iso-value）或密度值，表示这个点有多大程度在土地里或暴露在空气中。我们将讨论的是如何生成这个Voxel Grid，至于如何将它转化成可渲染的三角形网格则请读者参考Introduction of Voxels in Games Part I [4]。

其实山洞和悬空结构类似，把函数结果反过来理解就是了。

为了和高度图地形结合起来，我们首先把高度图数据转换成Voxel数据，然后再把3D分形函数的结果混合进去，例如在地形中混合山洞：

// t1为地形高度图对应的iso-value，t2为山洞分形函数生成的iso-value

float MultiplyIsoValue(float t1, float t2)

{

if (t1 > 0) {

return t2 > 0 ? sqrt(t1 * t2) : t2;

}

else {

return t2 > 0 ? t1 : -sqrt(t1 * t2);

}

}

5. 土地类型

除了山洞外，3D分形函数还可以用来计算不同的土地元素类型，例如沙漠、泥土、岩石等。现实中的土地种类分布大概是这样的：

底层主要是岩石，表面是泥土、沙漠或草原（我们暂且把草也当作土地的一种类型：），高处的地方可能会有雪地，某些低洼的地带可能会有沼泽。实际上要生成丰富逼真的土地元素是个很困难的事情，因为现实中这些都是复杂的自然现象不停地演绎迭代的结果，而我们的地形生成算法只能得到本地的、局部的信息。为此我们必须得加一些限制并简化一下模型：

  • 土地元素的类型是连续的，在[0, 1]之间
  • 最小值0为最底层的岩石，最大值1为最高处的雪地
  • 可在(0,1)中间定义若干种其它类型，如泥土、沙漠

这样，给定一个土地类型值e，我们可以通过插值知道它究竟什么类型，或者说介于哪两种类型之间，这样做的好处是：

  • 只需要一个3D分形函数即能求出土地类型
  • 生成出来的土地类型在空间上的分布过渡比较自然
  • 渲染时，土地类型e可以直接用来进行纹理插值

而这种简化最大的坏处就是牺牲了自由度，例如雪地永远不可能出现在岩石的附近，并且由于土地类型e不是离散的，对于游戏性的设计可能会有问题。

最后，用来生成土地类型的3D分形函数还需要考虑当前位置的地理高度，使得底层的地方更有可能出现岩石，高处的地方更有可能出现雪地：

6. 其它

除了上述所提到的地形元素外，一幅完整的地图还包括植被、河流等等，作者在这些方面的研究经验还不足，在此谨简单地罗列一下所了解到的一些相关的文档以飨读者。

  • Modeling of ecosystems as a data source for real-time terrain rendering [6]，这篇文档主要讲述如何实时在地形上放置植被以模拟自然的生态系统。
  • Erosion and Hydrology[7]

这个网页简单地讨论了几种河流的生成方法。

  • Polygonal Map Generation for Games[8]

这篇比较经典的Blog里的一些思想非常有用，介绍了从2D地形、河流到生态系统的生成。

  • Mandelbulb 3D[9]

对于喜欢分形的同学，Mandelbulb 3D是个不可错过的软件。



图7. Mandelbulb 3D[9]


8. 总结

作者认为未来的大型游戏中将会越来越多地出现程序化生成和用户自创建的内容，事实上目前市场上已经有不少游戏以此为卖点获得了成功。其实不仅是地形，包括树木、房屋、城市、声音等都可以借助程序化来生成，并且已经有不少研究成果。最后希望本文介绍的内容能引其读者对这方面的兴趣，并有机会一起探讨。
